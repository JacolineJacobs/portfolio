Second Chapter |

A new chapter for pre-loved books

Project Overview

Second Chapter is a responsive web project created to breathe new life into secondhand books and provide a welcoming space for book lovers.

Focusing on two key pages—the dynamic homepage and the interactive cart page—this project allowed me to practice responsive design and JavaScript interactivity.

By blending thoughtful design choices with custom JavaScript, Second Chapter delivers a seamless user experience that’s both functional and visually inviting.

Project Goals

The primary goal for Second Chapter was to strengthen my skills in responsive design while introducing JavaScript functionality to enhance user engagement. I set out to create:

  • A Mobile-First, Responsive Homepage:

    Ensuring an inviting layout that adapts beautifully across all devices, making browsing easy for everyone.

  • An Interactive Cart Page:

    Developing JavaScript functionality to create an intuitive, smooth cart experience for users.

  • Engaging Visuals and Easy Navigation:

    A straightforward, user-centered layout supported by a warm color palette, accessible typography, and effective browsing.

Design Process

Research

For Second Chapter, I explored several secondhand book websites to identify common design issues and areas for improvement. While I didn’t conduct formal user research, I observed the following problems that influenced my design decisions:

  • Cluttered Product Sections:

    Many websites were overloaded with text and information, making it hard for users to focus.

  • Inconsistent Layouts:

    Product cards and page elements varied greatly in size and style, causing confusion.

  • Mobile Responsiveness:

    Many sites had layouts that didn’t work well on mobile devices, making browsing difficult for users on the go.

By addressing these issues, I focused on creating a simpler, more intuitive browsing experience for Second Chapter, making it easier for users to navigate, find books, and engage with the platform.

Planning & Wireframes

Planning

Once I identified the key issues with existing secondhand book websites, I moved on to planning the layout and structure of Second Chapter. The goal was to create a clean, intuitive user experience that addressed these problems. I began by sketching out wireframes to establish the basic structure of the homepage, where key content would be prominently displayed. Key steps in the planning process:

  • Homepage Layout:

    The homepage needed to be simple yet engaging, with clear access to book categories, featured books, and the cart. I focused on easy navigation with a visible search icon and clearly defined sections for product categories.

  • Product Sections:

    I designed sections on the homepage to display different book categories and highlight featured books. Each section includes clean, consistent product cards with images, titles, and prices for easy browsing.

  • Mobile Layout:

    As a mobile-first design, I made sure the wireframes were adaptable to different screen sizes, ensuring the homepage and product sections would work seamlessly on mobile devices.
Wireframes

During the wireframing phase, I used Figma to sketch the layout and structure of the homepage. The goal was to focus on functionality and user flow before moving on to detailed design. Key aspects of the wireframes included:

  • Basic Layout:

    I mapped out the homepage with placeholders for the main product sections, search bar, and cart, ensuring easy navigation.

  • Mobile Responsiveness:

    The wireframes were designed with mobile users in mind, ensuring that the layout would adjust smoothly across different screen sizes.

  • Simple User Flow:

    I kept the flow intuitive, so users could easily browse product categories and quickly add books to their cart.

The wireframing stage allowed me to refine the layout and ensure a user-friendly structure before adding design details.

Colors & Typography

For Second Chapter, I carefully selected colors and fonts that would evoke a welcoming, modern feel while maintaining clarity and accessibility.

Colors:

  • Primary Color:

    A soft, earthy green (#7BAE7F) that gives a calm, vintage vibe—perfect for a secondhand book shop.

  • Light Color:

    A light, neutral tone (#ebeff0) to keep the background clean and easy on the eyes, offering good contrast with the primary and highlight colors.

  • Dark Color:

    A rich, deep charcoal (#01090a) for text and key elements, providing strong contrast and readability.

  • Highlight Color:

    A soft red (#FF6B6B) to draw attention to important buttons and calls to action, such as the "Add to Cart" button.

I also tested these colors for accessibility to ensure they would be clear and readable for all users.

Fonts:

  • Bitter:

    A serif font used for headings, providing a classic, literary feel that ties into the book theme.

  • Montserrat:

    A clean, modern sans-serif font used for body text, offering legibility and a contrast to the more decorative heading font.

The combination of these colors and fonts helps create a user-friendly, aesthetically pleasing design while maintaining accessibility standards.

Coding Skills Gained

In developing Second Chapter, I focused on building a solid foundation in both responsive design and JavaScript functionality to enhance the user experience. Here’s an overview of the key coding skills I applied and strengthened throughout the project:

    Responsive Design with HTML and CSS

  • Flexbox & Grid:

    Used to create a flexible layout that adjusts seamlessly across desktop and mobile views.

  • Media Queries:

    Applied CSS media queries to ensure that elements like product sections and navigation adapt smoothly to various screen sizes, enhancing the mobile-first experience.
    JavaScript Interactivity

  • Dynamic Cart Interaction:

    Implemented JavaScript to add and remove items in the cart, providing users with an interactive shopping experience

  • Responsive Navigation:

    Created a responsive menu using JavaScript to simplify navigation on smaller screens.

This project provided valuable hands-on experience with HTML, CSS, and JavaScript fundamentals, reinforcing my understanding of responsive web development and interactive features.

Conclusion

The Second Chapter project was an opportunity to apply my skills in responsive design and front-end development while focusing on creating an intuitive user experience for a secondhand book marketplace.

By addressing key usability issues—such as cluttered pages, inconsistent layouts, and lack of user feedback—I was able to create a more streamlined and engaging browsing experience.

The thoughtful use of color, typography, and mobile responsiveness ensured the site is both visually appealing and easy to navigate across all devices.

Overall, this project reinforced the importance of simplicity, accessibility, and user-centric design in building an effective online experience.
